# 🚀 shiko-prompt-assets

**Geminiと共に磨き上げる、高精度AIプロンプトの進化ログ**

このリポジトリは、noteクリエイター **shiko_nomad** によるプロンプト資産の保管庫です。単なる完成品の配布だけでなく、**「なぜそのプロンプトになったのか」**という改善プロセス（試行錯誤）を可視化し、再現性の高いAI活用を目指しています。

---

## 📋 コンセプト
AI（主にGemini）を活用し、「実益に直結するプロンプト」を開発しています。
- **透明性**: 失敗したプロンプトや改善指示の履歴を隠さず公開。
- **実用性**: note記事と連動し、即戦力として使えるテンプレートを提供。
- **共創**: Geminiとの対話を通じて、プロンプトの限界を突破するプロセスの記録。

## 🔗 関連リンク
- **note**: [shiko_nomad の記事一覧はこちら](https://note.com/shiko_nomad/all)
  - ここにあるプロンプトの具体的な活用事例や、マネタイズの戦略を詳しく解説しています。

## 📂 ディレクトリ構成
- `prompts/` : カテゴリ別のプロンプト本体（Markdown形式）。
- `drafts/` : 現在Geminiと改善中の実験的プロンプト。
- `assets/` : 解説用の画像や図解データ。

## 🛠 使い方
1. 各Markdownファイルにある「完成版プロンプト」をコピーします。
2. Gemini（またはお使いのLLM）に貼り付けて実行してください。
3. `[変数]` となっている箇所を、ご自身の状況に合わせて書き換えると効果的です。
